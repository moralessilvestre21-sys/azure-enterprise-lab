# Azure Monitor

## Objective

Monitor Azure resources, review activity logs, analyze performance metrics, and understand proactive alerting.

## Resources Monitored

- Virtual Machine
- Resource Group
- Storage Account

## Azure Monitor Features

- Metrics
- Activity Logs
- Alerts
- Insights
- Diagnostic Logs

## VM Metrics

Observed performance metrics including:

- CPU Usage
- Network In
- Network Out

These metrics help identify performance bottlenecks and resource utilization.

## Activity Log

The Activity Log records management operations performed against Azure resources.

Examples include:

- Resource creation
- Virtual machine start/stop
- RBAC changes
- Resource modifications

## Alerts

Configured an alert rule using CPU utilization as the monitored signal.

Example configuration:

- Signal: Percentage CPU
- Threshold: Greater than 80%
- Evaluation Frequency: Every 5 minutes

Alert rules help administrators proactively respond to performance or availability issues.
